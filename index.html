Angular Basics


Bootstrap Accordions


1. How do you define an AngularJS application?

An AngularJS application is created in a module. Inside that module are the different parts that make the application - controllers, services, filters, directives, etc. To designate the root element of that application you use the Angular directive 'ngApp', and place it near the root element of the page. The 'ngApp' directive will auto-bootstrap an AngularJS application

2. What is a controller?

A controller is an Object that lives inside the application and containes properties and functions that help control the flow of data in an application. Any controller can be attached to the DOM by using the 'ng-controller' directive; this directive will instantiate a new Controller object and create a new child scope that can be injected into the Controller constructor function using '$scope'

3. What does ng-model do?

The 'ng-model' directive binds the View into the Model. For example, it can be used to bind HTML form elements to a property on the scope. If the property does not already exist then it will be created.

4. What is interpolation markup?

Interpolation markup is part of AngularJS syntax. In AngularJS interpolation markup is used along with embedded expressions to provide data-binded to text nodes or attribute values. Interpolation markup is defined by the used of four curly braces, '{{}}'. AngularJS expressions are embedded inside these, i.e. '{{ Type in an expression }}'.

5. What is $scope?

$scope is an object that is used by the View to access the functions and properties of a specific controller. Only methods defined in the $scope object are accesible from the View.

6. What is dependency injection?

Dependency injection(DI) is a software pattern in order for components to get access to their needed dependencies. It basically provides an object with the dependencies(objects) it needs instead of having it construct them itself.

7. What is two-way binding?

Two-way data binding means that a property from the Model is bind to a HTML element in the View; when one of them is updated so will the other one. Data-binding is created using the 'ng-model' directive; it allows for a property in the Model to be changed directly from the View, and vice-versa.

8. What is 'controller as' syntax?

'Controller as' syntax allows us to define a named scope for a controller with the 'ng-controller' directive. With this new syntax we can define a property or method of a controller using 'this' instead of manipulation the $scope object at all. Using $scope along with nested controllers would create scope bleed; for that reason, 'Controller as' syntax is much clear in specifying the scope of each nested controller.

9. What is MV* architecture?

MV* refers to a Model-View-Wildcard design pattern. Some examples are, Model-View-Controller, Model-View-ViewModel, Model-View-Presenter,etc. These frameworks are great for solving certain problems, for example: Routing, Data-binding, Templates, Models, Data-access, etc.

10. What is the difference between a service and a factory?

Using a Factory gives much more power and flexibility, whereareas a Service is simply the "end result" of a Factory call. A Service is just a constructor, it's called with 'new', on the other hand a Factory is just a function that returns a value.
